🛡️ Defender for Cloud

Together we review the current security coverage in Azure and on on-premises servers. We verify that all desired Defender protection measures are actually active.

💡 Tip: In practice we often see that environments "appear protected", but essential components are missing or work differently than expected.
📊 Coverage overview: For a visual overview of Defender for Cloud coverage you can use the built-in Coverage workbook in Defender for Cloud (via Workbooks → Coverage), or the community build version on GitHub: github.com/rbaar/CloudCoverageWorkbook

📡 AMA Coverage

Check that the Azure Monitor Agent (AMA) is correctly deployed on all servers and that data collection rules are properly configured.

💡 Tip: AMA is the successor to the Legacy Log Analytics Agent (MMA). Check that the migration has been fully completed.

🕐 Retention

Check that the retention settings for logs and tables align with organisational requirements and compliance obligations.

💡 Tip: Default retention is 90 days. For incident analysis and compliance this is often insufficient. Check that critical tables have longer retention.
